    Untyped Arithmetic Expressions: 타입을 따지지 않는 언어 설계 - 수식

    less than 1 minute read

    Type System in Programming Languages

    타입은 프로그래밍 언어를 배워본 사람이라면 한 번은 들어봤을 개념이다. 주로 번역될 때 자료형이라는 용어를 쓰는데, ‘data type’을 ‘자료의 형태’라 번역한 것이라 보면 된다. 컴퓨터 프로그래밍의 범주에서 type은 보통의 경우 data type을 의미하기 때문에 틀린 번역은 아니지만, 표현의 명료함과 일반성을 위해 앞으로는 타입이라고 명명하겠다.

    그래서 타입이 뭘까? 이것부터 짚고 넘어가야 앞으로 배우게 될 내용을 명확히 이해할 수 있을 것이다. 아주 엄밀히 이야기하자면 너무 길어질 우려가 있으므로 간단하게만 생각해 보자. 나는 주로 타입을 해설할 때 ‘값의 집합’이라는 해석을 좋아한다.
